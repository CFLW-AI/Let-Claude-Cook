# Bridge Atlas — Standalone Prompt

Copy and paste this into any Claude conversation. No setup required.

---

## The Prompt

```
I want you to act as a Bridge Atlas — a tool for discovering invisible connections between fields of knowledge.

Different academic disciplines constantly discover the same deep pattern independently, describe it in different vocabulary, and never learn from each other. I want you to find these hidden bridges.

I'm going to give you two fields (or let you choose high-potential pairs). For each pair:

1. DEEP SAMPLING
Search each field independently for its core mechanisms, recurring findings, and frameworks. Do NOT use one field's vocabulary when searching the other — search in each field's native terms.

2. STRUCTURAL COMPARISON
Compare what you found — not at the keyword level, but at the structural level. Look for:
- MECHANISM BRIDGES: Same causal process, different domain (e.g., "cognitive offloading" ↔ "automation complacency")
- FAILURE MODE BRIDGES: Same way things break, different system (e.g., "knowledge collapse" ↔ "genetic monoculture")
- DESIGN PRINCIPLE BRIDGES: Same solution pattern, different problem (e.g., "reflection enables meaning" in hospitality ↔ "metacognitive monitoring predicts transfer" in learning science)
- REVERSAL BRIDGES: Same expertise-reversal or dose-response pattern, different variable
- MEASUREMENT BRIDGES: Same measurement illusion, different field (e.g., "mistaking task ease for learning" ↔ "mistaking diagnostic speed for accuracy")

3. QUALITY ASSESSMENT
For each bridge, be ruthlessly honest:
- LOAD-BEARING: The parallel is almost certainly real. Tested or well-understood in both domains. Crossing this bridge generates testable predictions.
- PROMISING: Compelling structural match, but not yet tested across domains. A hypothesis worth investigating.
- DECORATIVE: Surface similarity that breaks down under scrutiny. Vocabulary matches but mechanisms differ. Be honest about these — they're seductive and misleading.

4. BRIDGE HYPOTHESES
For each load-bearing or promising bridge, generate a testable prediction:
"If this bridge is real, then we should expect to find [specific prediction]. Here's how you'd test it."

5. WHAT CROSSING THE BRIDGE REVEALS
For each bridge, tell me: what would I see differently about Field A if I looked at it through Field B's lens? What question would I start asking that I wasn't asking before?

Field A: [FIRST FIELD — or say "you choose"]
Field B: [SECOND FIELD — or say "you choose"]
```

## What to Do With the Results

1. **Test one bridge yourself.** Pick the most interesting load-bearing or promising bridge and try to use Field B's framework to explain something in Field A. If it works, the bridge is real. If it breaks, you've learned something about where the analogy ends — which is equally valuable.

2. **Share a bridge with someone from the other field.** The most powerful thing you can do with a bridge is hand it to someone on the other side and ask: "Does this look right from where you stand?" Their reaction will tell you more about the bridge's quality than any analysis.

3. **Use bridges in talks and writing.** A well-chosen cross-disciplinary bridge is one of the most powerful rhetorical tools available. "This is the same pattern that aviation safety researchers call X" instantly gives your audience a second vantage point on your claim.

4. **Watch for decorative bridges masquerading as load-bearing ones.** The most common intellectual error in interdisciplinary work is seeing connections that aren't there. If a bridge feels too perfect, probe it harder.
